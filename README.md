这是一个用于分析 ML / AI 研究想法、论文和技术方案的小型研究分析 skill。

它关注的核心问题很简单：一个方法之所以复杂，是因为问题本身真的难，还是因为我们把问题建模得不够准确？

使用示例：

$research-target-reframing 分析这篇论文的研究 taste

$research-target-reframing 帮我判断这个 ML idea 是不是只是 trick

$research-target-reframing 用 why-not-simpler 的视角拆解这个方法

$research-target-reframing 把这个研究方向蒸馏成一个更清楚的 skill

基本立场

这个 skill 建立在一个很简单的判断上：trick 不一定是坏的，但 trick 是证据。
一个 trick 可能是问题本身要求的必要复杂性，也可能只是优化补丁、历史遗留物，或者为了 benchmark 调出来的经验做法。
好的重构不一定要消灭所有复杂性，但它至少应该解释：为什么原来的方法需要这些复杂性，以及有没有办法降低其中某些复杂性的中心地位。
我最感兴趣的研究，通常不是把方法堆得越来越复杂，而是让原本很别扭的行为变得自然。
